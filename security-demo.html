üö´ Old Insecure Method

‚ùå DEPRECATED - SECURITY RISK

The old iframe method allowed anyone to copy and use embed codes without authorization:

<iframe src="https://formcraft.ai/embed.html?code=embed_abc123def456" width="100%" height="500" frameborder="0"></iframe>

Security Issues:

  • No domain restrictions - anyone can embed forms anywhere
  • Permanent codes - no expiration or regeneration
  • No subscription validation - works even with cancelled accounts
  • No rate limiting per form - vulnerable to abuse

‚úÖ New Secure Method

‚úÖ SECURE - RECOMMENDED

The new JavaScript embed method uses signed tokens and domain validation:

<script src="https://formcraft.ai/embed.js?id=123&key=eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."></script>

Security Features:

  • Domain whitelist - only authorized domains can load forms
  • Time-limited tokens - 1-hour expiration for security
  • Real-time subscription checks - inactive accounts blocked immediately
  • Enhanced rate limiting - per-form, per-IP, subscription-aware
  • Signed tokens - prevents tampering and unauthorized access

üîß Implementation Details

1. Token Generation

POST /api/forms/:id/generate-token Authorization: Bearer <user_token> Response: { "success": true, "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "expiresIn": "1h" }

2. Domain Configuration

PUT /api/forms/:id/allowed-domains Authorization: Bearer <user_token> Content-Type: application/json { "allowedDomains": [ "example.com", "*.mydomain.com", "app.company.com" ] }

3. Secure Form Loading

POST /api/forms/embed-secure Content-Type: application/json { "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "hostname": "example.com" }

4. Secure Form Submission

POST /api/forms/submit-secure Content-Type: application/json { "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "data": { "name": "John Doe", "email": "john@example.com", "message": "Hello world" }, "hostname": "example.com" }

üöÄ Migration Guide

To upgrade from the old insecure embed system to the new secure system:

  1. Configure Domain Whitelist: Add authorized domains in the form builder
  2. Generate Secure Token: Click "Generate Secure Embed Code" in the form builder
  3. Update Website: Replace old iframe tags with new script tags
  4. Test Functionality: Verify forms load and submit correctly
  5. Monitor Usage: Check analytics for any blocked attempts

Note: Old iframe embeds will continue to work during the transition period, but should be migrated for enhanced security.
